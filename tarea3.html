Tarea 3 – Actualizaciones en Windows 10

Ejercicio 1

Deshabilitar las actualizaciones automáticas en Windows 10 Home

En este ejercicio se documentó un procedimiento para que un usuario sin conocimientos técnicos pueda deshabilitar las actualizaciones automáticas en Windows 10 Home. Se presentaron tanto métodos gráficos orientados al usuario como procedimientos técnicos que deshabilitan efectivamente el servicio.

1. Acceso a Windows Update

Desde el menú Inicio se accedió a Configuración → Actualización y seguridad.

Captura: Panel principal de Windows Update.

2. Pausar actualizaciones (Método usuario cero)

Windows 10 permite pausar temporalmente las actualizaciones automáticas.

Captura: Botón para pausar las actualizaciones.

3. Método técnico desde servicios

Para deshabilitar el servicio se utilizó la consola:

Win + R → services.msc

Captura: Consola de servicios del sistema.

Se seleccionó el servicio Windows Update (wuauserv) y se accedió a propiedades.

Captura: Propiedades del servicio Windows Update.

Para deshabilitar de forma efectiva se aplicó:

  • Detener el servicio
  • Cambiar Tipo de inicio a Deshabilitado

4. Opcional avanzado: Registro

Dado que Windows 10 Home no incluye gpedit.msc, la política puede aplicarse vía Registro.

HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU

Clave implicada:

NoAutoUpdate (DWORD) = 1

Captura: Clave del Registro utilizada para bloquear actualizaciones.

Conclusión

Combinando ambos métodos fue posible documentar el proceso completo para deshabilitar las actualizaciones automáticas en Windows 10 Home.

Ejercicio 2

Riesgos de deshabilitar las actualizaciones automáticas

Aunque deshabilitar las actualizaciones automáticas evita molestias temporales, presenta riesgos significativos desde el punto de vista de seguridad informática.

1. Exposición a vulnerabilidades

Las actualizaciones corrigen vulnerabilidades documentadas. Al no aplicarse, estas pasan de ser zero-day a vulnerabilidades conocidas, las cuales son las más explotadas mediante malware, ransomware y ataques remotos.

2. Incompatibilidad de software

Aplicaciones modernas requieren versiones actualizadas del sistema y librerías. La falta de parches puede impedir la ejecución correcta de navegadores, VPNs corporativas, suites de ofimática o servicios bancarios.

3. Incumplimiento normativo

En entornos corporativos y educativos las actualizaciones son obligatorias para cumplir estándares como ISO/IEC 27001, GDPR, ENS, PCI-DSS o HIPAA.

Conclusión

Desde el punto de vista operativo y de seguridad, deshabilitar las actualizaciones automáticas no resulta recomendable.

Ejercicio 3

Script PowerShell para habilitar actualizaciones automáticas

A continuación se presenta un script en PowerShell que revierte los cambios anteriores y habilita las actualizaciones automáticas de Windows 10.

Script solicitado

Set-Service -Name wuauserv -StartupType Automatic
Start-Service -Name wuauserv

Set-Service -Name UsoSvc -StartupType Automatic
Start-Service -Name UsoSvc

Remove-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" `
-Name "NoAutoUpdate" -ErrorAction SilentlyContinue

Explicación de comandos

  • Set-Service: configura el tipo de inicio del servicio.
  • Start-Service: inicia el servicio inmediatamente.
  • UsoSvc: servicio Update Orchestrator.
  • Remove-ItemProperty: elimina claves que bloquean el mecanismo de actualización.

Reversibilidad (opcional)

Si se quisiera convertir el script en reversible para deshabilitar de nuevo las actualizaciones: 

Set-Service -Name wuauserv -StartupType Disabled
Stop-Service -Name wuauserv -Force

Set-Service -Name UsoSvc -StartupType Disabled
Stop-Service -Name UsoSvc -Force

New-Item -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" -Force | Out-Null
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" `
-Name "NoAutoUpdate" -Value 1 -Type DWord
